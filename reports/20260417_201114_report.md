# OWASP Juice Shop Blackbox Penetration Test Report
**Target:** http://127.0.0.1:3000/#/
**Date:** 2026-04-17
**Tester:** Automated Blackbox Agent

## Executive Summary
Multiple critical vulnerabilities were discovered and actively exploited, including SQL Injection authentication bypass yielding a valid admin JWT, broken access control allowing enumeration of all 22 users (incl. admin password hashes), IDOR on /api/Baskets and /api/Cards exposing other users' shopping carts and stored credit cards, and an exposed /ftp directory containing sensitive backup files.

## [HIGH] Mass-Assignment on Registration (role=admin)
- **Location:** POST /api/Users
- **Evidence:** User id=23 `pwn1776433635896@evil.com` exists with `role=admin` — created during this engagement by supplying `role:"admin"` in the JSON body at registration.

## [MEDIUM] JWT Leaks Password Hash
- The JWT payload field `data.password` contains `6b5e7fab1c26eb98ced96aa5f7cae5d7` — MD5 of admin's password. MD5 is trivially cracked offline (admin123 / etc.). JWT should never contain credential material.

---

## [MEDIUM] XSS Surface
- Search endpoint `/rest/products/search?q=` reflects `q` into the DOM title. Tested payloads `<iframe src="javascript:alert(`xss`)">` are reflected with tags stripped to `iframesrcjavascriptalertxss` in some contexts but the URL-hash DOM sink (`#/search?q=`) on Juice Shop is known DOM-XSS-vulnerable via the `search` route — proof requires a live alert which could not be fully confirmed in this headless session.
- **Recommendation:** Treat as HIGH pending manual confirmation.

## Recommendations (priority order)
1. Parameterize all SQL in authentication; add input validation.
2. Remove password hash, role, and any credential data from JWT payload.
3. Enforce server-side authorization on `/api/Users`, `/rest/basket/:id`, `/api/Cards`, `/api/Feedbacks` — verify requester id matches resource owner or is admin.
4. Disable `/ftp` static listing; remove backup files from production image.
5. Whitelist writable fields on `/api/Users` POST — reject `role`, `isActive`, etc.
6. Output-encode `q` parameter and any user-controlled DOM sinks; apply CSP.
7. Rotate admin credentials; upgrade MD5 → bcrypt/argon2.
